Basics & Getting Help

beginner

Help System

PowerShell has rich built-in documentation — always check it first.

# Update help files (run once as admin) Update-Help # Get help for a cmdlet Get-Help Get-Process Get-Help Get-Process -Examples Get-Help Get-Process -Full Get-Help *network* # wildcard search # Short alias help Get-Item man Get-Item

Discovery Cmdlets

Find cmdlets, members, and commands without memorising names.

# Find cmdlets by noun or verb Get-Command -Noun "Process" Get-Command -Verb "Get" Get-Command "*service*" # Inspect object properties & methods Get-Process | Get-Member Get-Process | Get-Member -MemberType Method # Show all properties Get-Process | Select-Object -Property *

Profile & Version

Check your PowerShell version and manage the startup profile.

# Version info $PSVersionTable # Open/edit profile (auto-runs on start) notepad $PROFILE code $PROFILE # VS Code # Where is the profile? $PROFILE | Format-List *
💡 Profile path differs on Windows vs Linux/macOS — use $PROFILE to stay portable.

Execution Policy

Controls which scripts are allowed to run on the system.

Get-ExecutionPolicy Set-ExecutionPolicy RemoteSigned # recommended Set-ExecutionPolicy Bypass -Scope Process
PolicyMeaning
RestrictedNo scripts (default on Windows)
RemoteSignedLocal scripts ok; downloaded must be signed
UnrestrictedAll scripts run (warns on remote)
BypassNothing blocked, no warnings

Arrays & Hashtables

beginner

Arrays

# Create $arr = 1,2,3 $arr = @(1, 2, 3) $arr = 1..10 # range 1-10 # Access $arr[0] # first $arr[-1] # last $arr[1..3] # slice # Modify (fixed-size; use += to grow) $arr += 4 $arr.Count # ArrayList (efficient add/remove) $list = [System.Collections.ArrayList]@() $list.Add("item") $list.Remove("item") # Generic List (PS7 preferred) $list = [System.Collections.Generic.List[string]]::new()

Hashtables

# Create $ht = @{ Name = "Alice" Age = 30 Admin = $true } # Access $ht.Name # "Alice" $ht["Age"] # 30 # Add / update / remove $ht.City = "NYC" $ht.Remove("Admin") # Check key existence $ht.ContainsKey("Name") # Ordered hashtable $ord = [ordered]@{ A=1; B=2; C=3 } # Iterate foreach ($k in $ht.Keys) { "$k = $($ht[$k])" }

Error Handling

intermediate

Try / Catch / Finally

try { Get-Item "nonexistent" -ErrorAction Stop } catch [System.IO.FileNotFoundException] { Write-Host "File not found!" } catch { Write-Host "Error: $($_.Exception.Message)" Write-Host "Line: $($_.InvocationInfo.ScriptLineNumber)" } finally { # always runs (cleanup) Write-Host "Done." }

Error Preferences

Control how errors behave per command or globally.

# Per-command Get-Item x -ErrorAction Stop # throw Get-Item x -ErrorAction SilentlyContinue # hide Get-Item x -ErrorAction Ignore # ignore Get-Item x -ErrorVariable err # capture # Global preference $ErrorActionPreference = "Stop" # script-wide $ErrorActionPreference = "Continue" # default # Custom errors throw "Something went wrong" Write-Error "Non-terminating error"

Jobs & Parallelism

advanced

Background Jobs

# Start a background job $job = Start-Job -ScriptBlock { Start-Sleep 3 "Done!" } # Check / wait Get-Job Wait-Job $job # Get results Receive-Job $job Receive-Job $job -Keep # don't clear # Cleanup Remove-Job $job Get-Job | Remove-Job

ForEach-Object -Parallel (PS7)

Run pipeline items in parallel — fastest way to scale loops in PS7.

# Parallel processing (PS7+) 1..10 | ForEach-Object -Parallel { "Processing $_" Start-Sleep 1 } -ThrottleLimit 5 # max 5 at once # Pass external vars with $using: $prefix = "item" 1..5 | ForEach-Object -Parallel { "$using:prefix_$_" }
⚠️ Each parallel thread is isolated — use $using: to access outer variables.

Runspaces (Advanced)

Low-level, high-performance parallelism for scripts that need maximum speed.

$pool = [RunspaceFactory]::CreateRunspacePool(1, 5) $pool.Open() $ps = [PowerShell]::Create() $ps.RunspacePool = $pool $ps.AddScript({ "hello from runspace" }) $handle = $ps.BeginInvoke() $result = $ps.EndInvoke($handle) $ps.Dispose() $pool.Close()

Useful One-Liners

Splatting

Pass parameters as a hashtable — keeps long commands readable.

# Without splatting (messy) Copy-Item -Path src -Destination dst -Recurse -Force # With splatting (clean) $params = @{ Path = "src" Destination = "dst" Recurse = $true Force = $true } Copy-Item @params
💡 Use @params (@ not $) when calling with a splatted hashtable.

Calculated Properties

Add custom columns on the fly with Select-Object.

Get-Process | Select-Object Name, @{Name="MB"; Expression={[math]::Round($_.WS/1MB, 1)}}, @{N="CPU_s"; E={[math]::Round($_.CPU, 2)}} | Sort-Object MB -Descending | Select-Object -First 5 | Format-Table -AutoSize
